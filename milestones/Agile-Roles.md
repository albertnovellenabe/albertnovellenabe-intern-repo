# Agile Roles & Collaboration Reflection

## Reflection

### My Role in the Process
My primary role is as a **Developer**. In the Agile process, I contribute by breaking down technical tasks (like setting up GraphQL resolvers) into manageable pieces and providing realistic estimates on how long they will take. I also act as a "Technical Entrepreneur," suggesting ways to simplify code to deliver value to the user faster.

### Interacting with Other Roles
My work overlaps heavily with **QA** and **Product Management**. 
* With the **PM**, I discuss if a feature is technically feasible within our stack. 
* With **QA**, I provide unit tests and documentation so they know exactly what my code is supposed to do, which reduces the "back-and-forth" during bug fixing.
